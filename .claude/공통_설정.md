## [01] 공통 설정 — Spring Boot 마이그레이션

아래 NestJS 백엔드를 Spring Boot로 완전히 재구현해줘. 이 파일은 전체 프로젝트의 공통 설정이야. 실제 API는 다른 파일에서 이어서 구현할 거야.

---

## 기술 스택

- Java 17
- Spring Boot 3.x
- Spring Security 6.x
- Spring Data JPA + Hibernate
- MySQL 8.x
- JWT (jjwt 0.12.x)
- Gradle (Kotlin DSL)
- 기본 패키지: `com.vitaltrip`

---

## 데이터베이스 스키마

```sql
CREATE TABLE users (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(255) NOT NULL UNIQUE,
    name VARCHAR(100) NOT NULL,
    password VARCHAR(255),
    google_id VARCHAR(255) UNIQUE,
    birth_date VARCHAR(10) NOT NULL,
    country_code VARCHAR(10) NOT NULL,
    phone_number VARCHAR(20) NOT NULL,
    profile_image_url VARCHAR(500),
    provider ENUM('LOCAL','GOOGLE') NOT NULL DEFAULT 'LOCAL',
    role ENUM('USER','ADMIN') NOT NULL DEFAULT 'USER',
    refresh_token TEXT,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## 공통 응답 형식

모든 API 응답은 아래 형식으로 통일.

성공:
```json
{ "message": "성공", "data": { ... } }
```

실패:
```json
{ "message": "오류 설명", "data": null, "errorCode": "ERROR_CODE" }
```

`GlobalExceptionHandler` (`@RestControllerAdvice`)로 모든 예외를 위 형식으로 변환해야 함.

## 보안 구현 요구사항

1. **비밀번호 해싱**: BCryptPasswordEncoder (strength 12)
2. **JWT (jjwt, HS256)**:
    - Access Token payload: `{ sub: userId, email: userEmail }`
    - Access Token 만료: JWT_EXPIRES_IN ("7d" → 7일로 파싱)
    - Refresh Token 만료: JWT_REFRESH_EXPIRES_IN ("30d" → 30일)
    - Temp Token (Google OAuth 신규 가입): 10분 만료 (하드코딩)
3. **Spring Security 경로 설정**:
    - 인증 불필요: `/api/auth/**`, `/api/oauth2/**`, `/api/location/**`, `/api/first-aid/**`
    - JWT 인증 필요: `/api/user/**`
    - JWT 인증 + ADMIN role 필요: `/api/admin/**`
4. **CORS**:
    - development: `*`
    - production: `ALLOWED_ORIGIN` 환경변수
    - 허용 메서드: GET, POST, PUT, DELETE, PATCH
    - 허용 헤더: Content-Type, Authorization
5. **쿠키**: 관리자 토큰만 HttpOnly 쿠키 사용 (일반 사용자는 Bearer 토큰)

## 전역 설정

- 모든 API prefix: `/api`
- 입력값 검증: `@Valid` + `MethodArgumentNotValidException` → `VALIDATION_FAILED` 에러로 변환
