## [04] 관리자 API — Spring Boot 마이그레이션

공통 설정(01_공통_설정.md)이 이미 구현되어 있다고 가정하고, ADMIN API를 구현해줘.

공통 응답 형식:
- 성공: `{ "message": "성공", "data": { ... } }`
- 실패: `{ "message": "...", "data": null, "errorCode": "ERROR_CODE" }`

## ADMIN API

### 전체 사용자 목록 조회

```
GET /api/admin/users?page=0&size=20
Authorization: Bearer {accessToken}

인증 필요: JWT Bearer 토큰 + role = ADMIN

Query Parameters:
- page (선택, 기본 0): 0부터 시작하는 페이지 번호
- size (선택, 기본 20): 페이지 크기, 1~100 (범위 초과 시 자동 clamp)

Response 200:
{
  "message": "성공",
  "data": {
    "content": [
      {
        "id": 1,
        "email": "user@example.com",
        "name": "홍길동",
        "birthDate": "2000-01-01",
        "countryCode": "KR",
        "phoneNumber": "+821012345678",
        "profileImageUrl": null,
        "provider": "LOCAL",
        "providerId": null,
        "role": "USER",
        "createdAt": "2024-01-01T00:00:00.000Z",
        "updatedAt": "2024-01-01T00:00:00.000Z"
      }
    ],
    "page": 0,
    "size": 20,
    "totalElements": 100,
    "totalPages": 5,
    "first": true,
    "last": false,
    "hasContent": true,
    "hasNext": true,
    "hasPrevious": false
  }
}

Error:
- UNAUTHORIZED(401): 토큰 없음 또는 유효하지 않음
- FORBIDDEN(403): role != ADMIN

처리:
- Spring Data JPA Pageable 사용
- size clamp: Math.min(Math.max(size, 1), 100)
- providerId: 현재 User 엔티티에 없으므로 null 고정
```

### 관리자 권한 확인

```
GET /api/admin/me
Authorization: Bearer {accessToken}  (선택 — 없어도 200 반환)

인증 선택적: 토큰 없거나 유효하지 않으면 예외 발생 없이 isAdmin: false 반환

Response 200:
{ "message": "성공", "data": { "isAdmin": true } }

처리:
- OptionalJwtAuth: SecurityContext에서 Authentication 꺼내서 null이거나 anonymous면 isAdmin=false
- 인증된 경우: User.role == ADMIN이면 isAdmin=true, 아니면 false
- 어떤 경우에도 예외(4xx) 발생 없이 200 반환
```

---

위 ADMIN API 2개를 구현해줘.
구현 파일: AdminController, AdminService (필요시), 관련 DTO 전부.
관리자 인증은 `/api/auth/admin/login` 엔드포인트(02_인증_유저.md에서 구현)에서 발급된 JWT Bearer 토큰으로 처리.
