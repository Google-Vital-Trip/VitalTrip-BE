## [02] 인증·유저 API — Spring Boot 마이그레이션

공통 설정(01_공통_설정.md)이 이미 구현되어 있다고 가정하고, AUTH API와 USER API를 구현해줘.

공통 응답 형식:
- 성공: `{ "message": "성공", "data": { ... } }`
- 실패: `{ "message": "...", "data": null, "errorCode": "ERROR_CODE" }`

---

## AUTH API

### 1. 이메일 중복 확인

```
GET /api/auth/check-email?email=user@example.com

Response 200:
{ "message": "성공", "data": { "available": true } }

Error: VALIDATION_FAILED(400)
```

---

### 2. 회원가입

```
POST /api/auth/signup
Content-Type: application/json

Request:
{
  "email": "user@example.com",    // @Email @NotBlank
  "name": "홍길동",               // @NotBlank @Size(1,100)
  "password": "Pass1234!",        // 8~64자, 영문·숫자·특수문자(!@#$%^&*) 각 1자 이상
  "passwordConfirm": "Pass1234!", // password와 일치해야 함
  "birthDate": "2000-01-01",      // YYYY-MM-DD
  "countryCode": "KR",            // @Size(2,10)
  "phoneNumber": "+821012345678"  // E.164: +로 시작, 최대 15자리
}

Response 201: { "message": "성공", "data": null }

Error:
- VALIDATION_FAILED(400): 형식 오류 또는 passwordConfirm 불일치
- EMAIL_ALREADY_EXISTS(409): 이미 가입된 이메일

처리:
- BCrypt(strength=12)로 비밀번호 해싱
- User 저장 (provider=LOCAL, role=USER)
```

---

### 3. 로그인

```
POST /api/auth/login
Content-Type: application/json

Request:
{ "email": "user@example.com", "password": "Pass1234!" }

Response 200:
{
  "message": "성공",
  "data": {
    "accessToken": "eyJ...",
    "refreshToken": "eyJ...",
    "user": { "id": 1, "email": "user@example.com", "name": "홍길동" }
  }
}

Error:
- INVALID_CREDENTIALS(401): 이메일 없음, 비밀번호 불일치
- INVALID_CREDENTIALS(401): provider=GOOGLE 계정 (소셜 로그인 계정은 비밀번호 로그인 불가)

처리:
- accessToken, refreshToken 생성
- refreshToken을 DB(users.refresh_token)에 저장
```

---

### 4. 로그아웃

```
POST /api/auth/logout
Authorization: Bearer {accessToken}

Response 200: { "message": "성공", "data": null }

Error: UNAUTHORIZED(401)

처리: DB의 users.refresh_token을 null로 업데이트
```

---

### 5. 토큰 갱신

```
POST /api/auth/refresh
Content-Type: application/json

Request: { "refreshToken": "eyJ..." }

Response 200:
{ "message": "성공", "data": { "accessToken": "eyJ..." } }

Error: UNAUTHORIZED(401)

처리:
- refreshToken JWT 검증 (서명, 만료)
- DB에 저장된 refresh_token과 비교 (불일치 시 UNAUTHORIZED)
- 새 accessToken 발급
```

---

### 6. 비밀번호 변경

```
PUT /api/auth/password
Authorization: Bearer {accessToken}
Content-Type: application/json

Request:
{
  "currentPassword": "OldPass1!",
  "newPassword": "NewPass1!",       // 8~64자, 대소문자·숫자·특수문자 각 1자 이상
  "newPasswordConfirm": "NewPass1!"
}

Response 200: { "message": "성공", "data": null }

Error:
- VALIDATION_FAILED(400): newPassword != newPasswordConfirm
- VALIDATION_FAILED(400): provider=GOOGLE 계정 (소셜 계정은 비밀번호 변경 불가)
- INVALID_CREDENTIALS(401): currentPassword 불일치
```

---

### 7. Google OAuth2 로그인 URL 조회

```
GET /api/oauth2/login-url

Response 200:
{
  "message": "성공",
  "data": {
    "googleLoginUrl": "https://accounts.google.com/o/oauth2/v2/auth?client_id={GOOGLE_CLIENT_ID}&redirect_uri={GOOGLE_CALLBACK_URL}&response_type=code&scope=openid email profile&access_type=offline"
  }
}

처리: GOOGLE_CLIENT_ID, GOOGLE_CALLBACK_URL 환경변수로 URL 조합
```

---

### 8. Google OAuth2 콜백

```
GET /api/auth/google/callback?code={code}&error={error}

응답 없이 Redirect만 수행.

케이스 1 — error 파라미터 있거나 code 없음:
  redirect → {FRONTEND_URL}/auth/callback?error=true&errorCode=OAUTH_ERROR&message=...

케이스 2 — 기존 사용자 (google_id 또는 email로 DB 조회):
  DB의 google_id 업데이트 (없으면), refresh_token 저장
  redirect → {FRONTEND_URL}/auth/callback?success=true&accessToken=...&refreshToken=...&email=...&name=...&profileImageUrl=...

케이스 3 — 신규 사용자 (DB에 없음):
  tempToken 발급 (10분 만료 JWT)
  redirect → {FRONTEND_URL}/auth/callback?needsProfile=true&tempToken=...&email=...&name=...&profileImageUrl=...

케이스 4 — 서버 오류:
  redirect → {FRONTEND_URL}/auth/callback?error=true&errorCode=OAUTH_ERROR&message=...

Google 사용자 정보 가져오는 방법:
1. POST https://oauth2.googleapis.com/token
   Body: code, client_id, client_secret, redirect_uri, grant_type=authorization_code
   → access_token 획득

2. GET https://www.googleapis.com/oauth2/v3/userinfo
   Header: Authorization: Bearer {access_token}
   → { sub, email, name, picture } 반환

   sub    → google_id
   email  → email
   name   → name
   picture → profileImageUrl
```

### 9. 관리자 로그인

```
POST /api/auth/admin/login
Content-Type: application/json

Request: { "email": "admin@example.com", "password": "Pass1234!" }

Response 200:
- 쿠키 설정 (httpOnly, secure, sameSite=strict):
  adminAccessToken  (maxAge: 3600초 = 1시간)
  adminRefreshToken (maxAge: 2592000초 = 30일)
- Body: { "message": "어드민 로그인이 완료되었습니다", "data": null }

Error:
- INVALID_CREDENTIALS(401): 이메일 없음 또는 비밀번호 불일치
- INVALID_CREDENTIALS(401): provider=GOOGLE 계정
- FORBIDDEN(403): role != ADMIN
```

---

### 10. 관리자 토큰 갱신

```
POST /api/auth/admin/refresh
Cookie: adminRefreshToken={token}

Response 200:
- 쿠키 갱신: adminAccessToken (새 토큰, maxAge: 3600초)
- Body: { "message": "토큰이 갱신되었습니다", "data": null }

Error:
- UNAUTHORIZED(401): 쿠키 없음 또는 토큰 유효하지 않음
- FORBIDDEN(403): role != ADMIN
```

## USER API

모든 엔드포인트: `Authorization: Bearer {accessToken}` 필수

### 1. 내 프로필 조회

```
GET /api/user/profile

Response 200:
{
  "message": "성공",
  "data": {
    "id": 1,
    "email": "user@example.com",
    "name": "홍길동",
    "googleId": null,
    "birthDate": "2000-01-01",
    "countryCode": "KR",
    "phoneNumber": "+821012345678",
    "profileImageUrl": null,
    "provider": "LOCAL",
    "role": "USER",
    "createdAt": "2024-01-01T00:00:00.000Z",
    "updatedAt": "2024-01-01T00:00:00.000Z"
  }
}

Error: UNAUTHORIZED(401)
```

---

### 2. 프로필 수정

```
PUT /api/user/profile
Authorization: Bearer {accessToken}
Content-Type: application/json

Request:
{
  "name": "홍길동",            // @NotBlank @Size(1,100)
  "birthDate": "2000-01-01",
  "countryCode": "KR",         // @Size(2,10)
  "phoneNumber": "+821012345678"
}

Response 200: { "message": "성공", "data": null }

Error: UNAUTHORIZED(401), VALIDATION_FAILED(400)
```
