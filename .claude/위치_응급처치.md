## [03] 위치·응급처치 API — Spring Boot 마이그레이션

공통 설정(01_공통_설정.md)이 이미 구현되어 있다고 가정하고, LOCATION API와 FIRST-AID API를 구현해줘.
두 API 모두 인증 불필요 (공개 엔드포인트).

공통 응답 형식:
- 성공: `{ "message": "성공", "data": { ... } }`
- 실패: `{ "message": "...", "data": null, "errorCode": "ERROR_CODE" }`

---

## LOCATION API

### 1. 좌표로 국가 식별

```
POST /api/location/identify-country
Content-Type: application/json

Request:
{
  "latitude": 37.5665,   // @DecimalMin("-90") @DecimalMax("90")
  "longitude": 126.978   // @DecimalMin("-180") @DecimalMax("180")
}

외부 API 호출:
GET https://api.bigdatacloud.net/data/reverse-geocode-client?latitude={lat}&longitude={lon}&localityLanguage=en
타임아웃: 5초
응답에서 추출: countryCode (ISO 3166-1 alpha-2), countryName

응급번호: 아래 정적 Map에서 countryCode로 조회 (미등록 시 DEFAULT 사용)

Response 200:
{
  "message": "성공",
  "data": {
    "countryCode": "KR",
    "countryName": "South Korea",
    "latitude": 37.5665,
    "longitude": 126.978,
    "emergencyContact": {
      "fire": "119",
      "police": "112",
      "medical": "119",
      "general": "112"
    }
  }
}

Error: EXTERNAL_SERVICE_UNAVAILABLE(503)
```

### 2. 주변 의료시설 검색

```
GET /api/location/nearby?latitude=37.5665&longitude=126.978&radius=5000&type=hospital&language=ko

Query Parameters:
- latitude  (필수): -90 ~ 90
- longitude (필수): -180 ~ 180
- radius    (선택, 기본 5000): 500 ~ 50000 (미터)
- type      (선택, 기본 hospital): hospital | pharmacy | emergency
- language  (선택, 기본 en): BCP 47 언어 코드

외부 API 호출:
POST https://places.googleapis.com/v1/places:searchNearby
Headers:
  X-Goog-Api-Key: {GOOGLE_PLACES_API_KEY}
  X-Goog-FieldMask: places.displayName,places.formattedAddress,places.nationalPhoneNumber,places.location,places.regularOpeningHours,places.websiteUri

Body:
{
  "includedTypes": {아래 매핑 참고},
  "maxResultCount": 20,
  "locationRestriction": {
    "circle": {
      "center": { "latitude": lat, "longitude": lon },
      "radius": radius (double)
    }
  },
  "languageCode": "{language}"
}

type별 includedTypes 매핑:
  hospital  → ["hospital", "doctor"]
  pharmacy  → ["pharmacy"]
  emergency → ["hospital"]

Google Places 응답 파싱 (places 배열 각 항목):
  displayName.text             → name        (null이면 빈 문자열)
  formattedAddress             → address     (없으면 null)
  nationalPhoneNumber          → phoneNumber (없으면 null)
  location.latitude            → latitude
  location.longitude           → longitude
  regularOpeningHours.openNow  → openNow     (없으면 null)
  regularOpeningHours.weekdayDescriptions → openingHours (없으면 null)
  websiteUri                   → websiteUrl  (없으면 null)

distance 계산:
  Haversine 공식 (R = 6,371,000 m)
  결과를 km 단위, 소수점 1자리로 반올림
  예: Math.round(haversineMeters / 100.0) / 10.0

결과 정렬: distance 오름차순

캐싱 (ConcurrentHashMap, 메모리):
  TTL: 2시간 (7,200,000 ms)
  캐시 키: "{type}:{String.format("%.2f", lat)}:{String.format("%.2f", lon)}:{radius}:{language}"
  예: "hospital:37.57:126.98:5000:ko"

타임아웃: 10초

Response 200:
{
  "message": "성공",
  "data": [
    {
      "name": "서울대학교병원",
      "address": "서울특별시 종로구 대학로 101",
      "phoneNumber": "02-2072-2114",
      "latitude": 37.58,
      "longitude": 126.99,
      "distance": 1.5,
      "openNow": true,
      "openingHours": ["Monday: 09:00 - 18:00"],
      "websiteUrl": "http://hospital.snuh.org"
    }
  ]
}

Error: EXTERNAL_SERVICE_UNAVAILABLE(503)
```

## FIRST-AID API

### 응급처치 AI 조언

```
POST /api/first-aid/advice
Content-Type: application/json

Request:
{
  "symptomType": "BLEEDING",
  "symptomDetail": "오른쪽 손목에서 출혈이 심하게 발생하고 있습니다.",
  "latitude": 37.5665,
  "longitude": 126.978
}

SymptomType enum:
  BLEEDING, BURNS, FRACTURE, ALLERGIC_REACTION, SEIZURE, HEATSTROKE,
  HYPOTHERMIA, POISONING, BREATHING_DIFFICULTY, ANIMAL_BITE, FALL_INJURY

검증: symptomDetail @Size(10, 2000), latitude -90~90, longitude -180~180

처리 흐름:
1. latitude/longitude → BigDataCloud API → countryCode, countryName
   (LOCATION API의 identify-country와 동일 로직, 서비스 공유 가능)
2. countryCode → EmergencyContacts 정적 Map → emergencyContact
3. OpenAI GPT-4o 호출 (아래 정확한 프롬프트 사용)

Response 200:
{
  "message": "성공",
  "data": {
    "content": "출혈 부위를 깨끗한 천으로 압박하세요.\n...",
    "summary": "직접 압박으로 출혈을 멈추세요.",
    "recommendedAction": "119에 즉시 신고하세요.",
    "identificationResponse": {
      "countryCode": "KR",
      "countryName": "South Korea",
      "latitude": 37.5665,
      "longitude": 126.978,
      "emergencyContact": { "fire": "119", "police": "112", "medical": "119", "general": "112" }
    },
    "disclaimer": "본 정보는 참고용이며 전문 의료 조언을 대체하지 않습니다.",
    "confidence": 92,
    "blogLinks": ["https://example.com/first-aid"]
  }
}

Error: AI_SERVICE_UNAVAILABLE(503)
```

## OpenAI 호출 상세

model: gpt-4o
temperature: 0.3
response_format: { type: "json_object" }

### system message (아래 문자열 그대로 사용)

```
You are an emergency medical expert providing first-aid guidance for travelers.

LANGUAGE DETECTION — use grammar patterns, NOT medical terms:
"I got burned on my hand" → English (I, got, on my)
"Me quemé en la mano" → Spanish (Me quemé, en la)
"뜨거운 물에 손을 데었어요" → Korean (뜨거운, 데었어요)
"J'ai été brûlé par de l'eau chaude" → French (J'ai été, par de)
Respond ENTIRELY in the detected language — never mix languages. ALL sections must be in the same language.

Respond ONLY with a JSON object:
{
  "content": "exactly 9-10 first-aid steps separated by \n, no numbers or bullets",
  "summary": "1-2 sentence overview",
  "recommendedAction": "single most urgent action right now",
  "disclaimer": "AI advice only — not professional diagnosis; seek immediate professional care; call emergency services",
  "confidence": <integer 0-100>,
  "blogLinks": ["trusted medical URL 1", "trusted medical URL 2", "trusted medical URL 3"]
}

Rules:
- content: exactly 9-10 plain-text lines separated by \n, no leading numbers or bullets
- If life-threatening, make calling emergency services the first step
- blogLinks must be real URLs from trusted medical/health organizations
- confidence: how reliably the provided info supports a good response (0-100)
```

### user message (아래 형식으로 동적 조합)

```
symptomType: {symptomType}
symptomDetail: {symptomDetail}
countryCode: {countryCode}
additionalGuidance: {아래 매핑에서 symptomType에 해당하는 문자열}
```

### symptomType별 additionalGuidance 매핑

```
BLEEDING:
  "Focus on controlling bleeding via direct pressure, elevation, and pressure points. Include severity assessment and shock prevention."

BURNS:
  "Focus on cooling the burn, preventing infection, and determining burn degree. Include removing from heat source and protecting the area."

FRACTURE:
  "Focus on immobilization and preventing further injury. Include splinting technique and supporting the injured area."

ALLERGIC_REACTION:
  "Focus on identifying and removing the allergen, managing symptoms. Include severe-reaction steps and when to use epinephrine."

SEIZURE:
  "Focus on protecting the person during the seizure and recovery position. Include timing the seizure and when to call emergency services."

HEATSTROKE:
  "Focus on rapid cooling and moving to shade/cool environment. Include monitoring consciousness and preventing shock."

HYPOTHERMIA:
  "Focus on gradual warming and preventing heat loss. Include proper positioning and avoiding rapid rewarming."

POISONING:
  "Focus on identifying the poison and appropriate decontamination. Include when NOT to induce vomiting and calling poison control."

BREATHING_DIFFICULTY:
  "Focus on positioning for easier breathing and clearing airways. Include recognizing severe respiratory distress."

ANIMAL_BITE:
  "Focus on wound cleaning, bleeding control, and infection prevention. Include rabies considerations and when to seek immediate care."

FALL_INJURY:
  "Focus on spinal injury precautions and assessing for fractures. Include when to move the person and head injury signs."
```

### AI 응답 파싱 필드

```
content          → String
summary          → String
recommendedAction → String
disclaimer       → String
confidence       → int (0-100)
blogLinks        → List<String>
```
